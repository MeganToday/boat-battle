# Boat Battle

A browser-based battleship-style game built with HTML, CSS, and JavaScript.

## Git & GitHub Workflow

After completing any meaningful unit of work — a new feature, a bug fix, a significant UI change — commit and push immediately. Never let work accumulate without a commit. Use clean, descriptive commit messages in the imperative mood (e.g., "Add ship placement grid", "Fix hit detection logic").

```bash
git add <files>
git commit -m "Short description of what changed"
git push origin main
```

This ensures:
- Work is never lost
- Progress is always visible on GitHub
- The public game link (GitHub Pages) stays up to date

## Project Structure

- `index.html` — main game entry point
- `style.css` — game styles (to be created)
- `game.js` — game logic (to be created)

## Tech Stack

- Vanilla HTML/CSS/JavaScript (no build step, no framework)
- Hosted via GitHub Pages for a public playable link
